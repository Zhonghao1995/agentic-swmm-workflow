# v0.7.0-alpha — Modeling memory, Claude Agent SDK provider, CLI/UX overhaul

**Released**: 2026-05-21
**Status**: Pre-release (alpha), built on top of v0.6.4. The default `pip install aiswmm` continues to deliver v0.6.4; this version requires opt-in.
**PyPI version string**: `0.7.0a1` (PEP 440)
**Git tag**: `v0.7.0a1`

## How to install

```bash
pip install aiswmm==0.7.0a1   # pin
pip install --pre aiswmm      # allow any pre-release
```

The Claude Agent SDK provider is an optional extra:

```bash
pip install "aiswmm[claude]==0.7.0a1"
```

## What's new

### Modeling-memory substrate

An on-disk memory layer under `memory/modeling-memory/`: `parametric_memory`
(run-level parameters and QA metrics), `calibration_memory` (accepted
calibrations), `reference_benchmarks` (library defaults) with a per-project
`project_overrides.yaml` overlay, a citation library, and `negative_lessons`
(known-bad parameter regions). Includes watershed-similarity matching and
SQLite indexing for large stores. Scaffold it with `aiswmm bootstrap memory`.

### Memory-informed runtime

The planner can read modeling memory to disambiguate ambiguous requests, adapt
QA thresholds to project history, and carry parameter priors across watersheds,
with a transparency log of which memory entries were used. Opt out per run with
`--ignore-memory`.

### Claude Agent SDK provider (optional)

A second LLM backend that routes the planner through a Claude Pro/Max
subscription via the local `claude` CLI, installed via the `aiswmm[claude]`
extra. The default OpenAI provider is unchanged and pulls none of this.

### CLI / UX overhaul

New verbs: `aiswmm compare`, `aiswmm storm` (Chicago / Huff / SCS design
hyetographs), `aiswmm trace`, `aiswmm uncertainty plan`, and `aiswmm bootstrap
memory`. A unified flag convention across every verb (`--inp` / `--json` /
`--quiet` / `--example`), grouped `--help` output, `error: / cause: / hint:`
messages, and an honesty layer that detects SWMM `ERROR` output instead of
reporting false success.

## Reproducibility

v0.6.4 remains the latest **stable** release. Every v0.6.4 pin —
`pip install aiswmm==0.6.4`, the `v0.6.4` Git tag, and the `:v0.6.4` Docker
image — is immutable and unaffected by this pre-release.

This is an alpha: the Claude Agent SDK provider in particular is new and not
yet exercised at scale. Please report issues on GitHub.
